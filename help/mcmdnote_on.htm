NOTE_ON


Command Format:

NOTE_ON:<Note name or number>,[octave],[volume]

What it does:

Starts playing the specified note but doesn't stops playing that note until you call NOFF or STOP. This command is similar to a, a#, c c# etc commands to play note except that the interval for how long note will be played is not specified. You can use this command to simultaneously play many notes within a single tune but it's time taking to calculate and stop the notes. See the examples.

Short Form:

  NON

Parameters:

Note name or number: Note name (i.e. a,a#,c,c# etc) or direct number of note you want to start.
Octave: In which octave to play the note. Also supports ++ etc operators. If note specified currently set octave is used.
Volume: Volume with which the note should be played. Also supports ++ etc operators. If note specified currently set volume is used.

Value constraints:

Note Number: 0-87 or a,a#,b,c,c#,d,d#,e,e#,f,g,g#
Octave: 0 - 7
Volume: 0-100

Remarks:

This command is equivalent to commands like a, a#, b. c# etc which plays the note specified by its name instead of number. There is no performance issue between two versions. In general playing a note by it's name is much more readable then it's number. However this command is useful when you want to write the script to play the note whose number is stored in a variable. Octave argument is optional but is desirable to improve readability. If change the octave before stopping note you must supply octave while turning the note off.

Examples:

1. Play note for interval = 10

INSTRUMENT:Flute
NOTE_ON:a#
WAIT_FOR:10
NOTE_OFF:a#

2. Play the notes a# in octave 3 and c# in octave 4 simultaneously. Insert some delays so that both could be distinguished.

INSTRUMENT:Flute
NOTE_ON:a#,3 'Start a#
WAIT_FOR:3
NOTE_ON:c#,4 'Start c#
WAIT_FOR:50
NOTE_OFF:a#,3 'Stop a#
WAIT_FOR:10
NOTE_OFF:c#,4 'Stop c#
