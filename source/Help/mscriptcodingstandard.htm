MScript Coding Standard

  1. Each MScript command except names of the note should be in capital.
  2. MScript variables should be in mixed case and should start with a capital letter.
  3. Labels should also be in mixed case and should start with a capital letter.
  4. Try to avoid writing instrument number instead of instrument name. For example,

    INSTRUMENT:Sitar

    is more preferable then.

    INSTRUMENT:104.

  5. MScript supports 3 styles of comments:

    'VB style comment supported
    --SQL Server style comment supported
    //C++ style comment supported.

    Try to use VB style comments as standard.
  6. MScript supports short names for almost all the commands. For example instead of writing,

    INSTRUMENT:Sitar

    you can write,

    I:Sitar

    This short names are supported to do trial and error quickly and should not be used in released version of your script.
